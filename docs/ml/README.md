# ML

## Что находится в `ml/`

- `ml/model` — код обучения, инференса, explainability, метрик и подготовки данных
- `ml/artifacts` — сериализованные model bundles (`*.joblib`)
- `ml/data/raw` — локальная копия исходного датасета
- `ml/reports` — generated reports, shortlist и артефакты оценки качества

## Почему артефакты вынесены в `ml/artifacts`

Артефакты модели логически относятся к ML-контуру, но не должны смешиваться с исходным кодом. Отдельная директория:

- не ломает инференс, потому что пути централизованы в `ml/model/utils.py`
- явно разделяет код и бинарные результаты
- упрощает будущий перенос артефактов во внешний storage

## Запуск обучения

```bash
python -m ml.model.main
```

Совместимый вариант:

```bash
python main.py
```

## Путь по умолчанию к модели

`ml/artifacts/best_model.joblib`
